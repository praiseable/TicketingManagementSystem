# UI Quality Pack 02 — Issue, Agile, Sprint, and Time Feedback

This pack improves validation, success/error feedback, loading states, and confirmations in the most active execution screens: issue list, issue detail, Kanban, backlog, sprints, worklogs, and timers.

## Acceptance Criteria

- Sub-task creation requires a title.
- Issue linking requires a target issue and blocks self-linking where supported.
- Attachment upload/delete/open failures are visible.
- Worklog create/edit/delete actions show validation and confirmation.
- Timer start/pause/resume/stop gives visible feedback.
- Backlog move-to-sprint requires a selected sprint.
- Sprint creation validates required name/date/capacity where supported.
- Sprint completion asks for confirmation.
- Bulk delete and dangerous actions are confirmed.
